vplotter turns a Raspberry Pi into a wall-hanging drawing machine. It reads g-code from standard tools like Inkscape's gcodetools extension, controls two stepper motors via EasyDriver boards to move a hanging gondola across the canvas, and uses a servo motor to raise and lower the pen. This documentation covers everything you need to build, configure, and run your own vplotter.

## How vplotter works

vplotter implements a V-plotter (also called a polargraph): two stepper motors hang at the top of a vertical surface, each winding a cord attached to the drawing gondola. By adjusting the cord lengths independently, the gondola swings to any XY position. A servo motor attached to the gondola lifts and lowers the pen.

<Steps>
  <Step title="Build the hardware">
    Mount two stepper motors (with EasyDriver boards) at the top of your canvas. Attach cords from each motor spool to the gondola holding your pen and servo.
  </Step>
  <Step title="Install and build vplotter">
    Clone the repository, install dependencies (`git`, `cmake`, `flex`, `wiringPi`), and build with CMake. See [Installation](/installation) for the full steps.
  </Step>
  <Step title="Measure your geometry">
    Measure the distance between the two motors (`baselength`), and the horizontal and vertical offsets from the left motor to your canvas origin (`x0`, `y0`). See [Geometry](/hardware/geometry) for details.
  </Step>
  <Step title="Generate g-code and plot">
    Use Inkscape with the gcodetools extension to convert artwork to g-code, then feed it to vplotter. See [Generating g-code](/reference/generating-gcode) for the workflow.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Standard g-code input" icon="file-code">
    Reads g-code compatible with Inkscape's gcodetools extension. Pipe any `.ngc` file directly to vplotter via stdin.
  </Card>
  <Card title="Linear and arc moves" icon="pen-line">
    Supports rapid moves (G0), linear moves (G1), clockwise arcs (G2), and counter-clockwise arcs (G3) with radius or center-point coordinates.
  </Card>
  <Card title="Servo pen control" icon="arrow-up-down">
    Z-axis commands (G0 Z1 / G0 Z0) lift and lower the pen via a PWM servo, giving clean start and end points for each drawn path.
  </Card>
  <Card title="Configurable geometry" icon="ruler">
    All machine dimensions are set at runtime via command-line flags — no recompilation needed when you change your canvas setup.
  </Card>
</CardGroup>
