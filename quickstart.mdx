description: "Go from a freshly installed vplotter to your first drawing: measure your canvas, test the connection, create a g-code file, and run the plotter."
---

This guide walks you through everything you need to make your first plot — from measuring your hardware to feeding g-code into vplotter and watching the gondola move.

## Prerequisites

Before you start, make sure you have:

- vplotter installed (see [Installation](/installation))
- Your hardware fully assembled: two stepper motors at the top of the canvas, cords attached to the gondola, and a servo mounted on the gondola

<Steps>
  <Step title="Measure your canvas geometry">
    vplotter needs four measurements to compute cord lengths correctly. Take these from your physical setup:

    | Flag | What to measure |
    |------|----------------|
    | `--baselength` | Distance in mm between the two motor axles |
    | `--x0` | Horizontal offset in mm from the left motor to your canvas origin |
    | `--y0` | Vertical offset in mm from the left motor to your canvas origin (negative — the origin is below the motors) |
    | `--steps` | Steps per mm for your stepper motors (depends on motor, driver microstepping, and spool diameter) |

    See [Hardware geometry](/hardware/geometry) for a detailed diagram and measurement guide.
  </Step>
  <Step title="Test the connection">
    Run vplotter with your measured values and no input file to verify that the motors and servo respond. Use the example values below, substituting your own measurements:

    ```bash
    sudo vplotter --x0=170 --y0=-720 --baselength=685 --z_up=11 --z_down=7 --steps=40.1
    ```

    **What each flag means:**

    - `--x0=170` — canvas origin is 170 mm to the right of the left motor
    - `--y0=-720` — canvas origin is 720 mm below the motors (negative Y)
    - `--baselength=685` — the two motors are 685 mm apart
    - `--z_up=11` — servo PWM value that lifts the pen
    - `--z_down=7` — servo PWM value that lowers the pen to the canvas
    - `--steps=40.1` — steps per mm of cord movement

    vplotter will wait for g-code on stdin. Press **Ctrl+C** to exit once you confirm the servo moved on startup.
  </Step>
  <Step title="Create a simple g-code file">
    Create a file called `myfile.ngc` with a minimal g-code program. This example lifts the pen, moves to a start position, draws a short horizontal line, and stops:

    ```gcode
    G21 (units in mm)
    G00 Z5 (lift pen)
    G00 X50 Y50 (move to start)
    G01 Z-0.125 F100 (lower pen)
    G01 X100 Y50 F400 (draw line)
    G00 Z5 (lift pen)
    M30
    ```

    Save the file on your Raspberry Pi in the same directory you will run vplotter from.
  </Step>
  <Step title="Run the plotter">
    Feed the g-code file to vplotter using stdin redirection:

    ```bash
    sudo vplotter --x0=170 --y0=-720 --baselength=685 --z_up=11 --z_down=7 --steps=40.1 < myfile.ngc
    ```

    vplotter reads each command from the file, computes the required cord lengths, steps the motors, and controls the servo. The gondola will move to X50 Y50, lower the pen, draw to X100 Y50, lift the pen, and stop when it reaches `M30`.
  </Step>
  <Step title="Stop the plotter">
    vplotter stops automatically when it reads an `M30` command (program end). You can also stop it at any time by pressing **Ctrl+C** in the terminal, or by typing `M30` and pressing Enter if you are running interactively without a file.
  </Step>
</Steps>

<Tip>
  See the [G-code reference](/usage/gcode-reference) for the full list of supported G and M codes, including arc moves (G2/G3), unit switching (G20/G21), and feed rate control.
</Tip>
